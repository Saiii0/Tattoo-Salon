Документ: Архитектура и структура приложения "Тату Салон"
Дата: 2026-03-13

1) Обзор
Приложение — SPA на React + TypeScript + Vite. Навигация реализована через React Router. Данные хранятся в памяти на клиенте и инициализируются мок‑данными. Глобальное состояние и бизнес‑логика объединены в AppContext (React Context). UI — Ant Design, стили страниц вынесены в отдельные CSS‑файлы.

2) Технологический стек и ключевые зависимости
- React 18 + TypeScript
- Vite (сборка/дев‑сервер)
- React Router v7 (маршрутизация)
- Ant Design (UI‑компоненты)
- Day.js (работа с датами)
- Дополнительные пакеты: @ant-design/icons, recharts (для графиков, если потребуется), и др.

3) Структура папок (src)
src/
  main.tsx             — точка входа: монтирует React и подключает глобальные стили
  styles/              — глобальные CSS (fonts.css, index.css, theme.css)
  app/
    App.tsx            — рендерит RouterProvider с конфигурацией роутов
    routes.tsx         — декларация маршрутов, корневой провайдер контекста
    types.ts           — общие типы доменной модели
    data/mockData.ts   — мок‑данные: users, services, reviews, orders, favorites
    context/
      AppContext.tsx   — глобальное состояние и бизнес‑логика
    components/
      Layout.tsx       — основной каркас приложения (sidebar, header, content)
      ProtectedRoute.tsx — защита приватных маршрутов
    pages/             — все страницы приложения + локальные CSS на каждую страницу

4) Точка входа и глобальные стили
Файл src/main.tsx:
- Импортирует App и глобальный CSS: src/styles/index.css
- Монтирует приложение в #root

Файл src/styles/index.css:
- Импортирует fonts.css и theme.css
- Содержит базовый reset: box-sizing, размеры html/body/#root

Файл src/styles/theme.css:
- Переменные дизайна (цвета, радиусы, размеры шрифтов)
- Базовая типографика (h1-h4, label, button, input)

5) Маршрутизация
Файл src/app/routes.tsx:
- Создает router через createBrowserRouter
- Корневой уровень оборачивает все маршруты в AppProvider (Root -> AppContext)
- Публичные маршруты:
  /login — вход
  /register — регистрация
- Приватный корень "/" защищен через ProtectedRoute:
  /            — Home
  /favorites   — Favorites
  /service/:id — ServiceDetail
  /profile     — Profile
  /manage-services — ManageServices (admin)
  /user-management — UserManagement (admin)
  /orders      — Orders (master)
  /my-orders   — MyOrders (user)
  /master-stats — MasterStats (master)
  /admin-stats  — AdminStats (admin)
  *            — NotFound

Почему так:
- ProtectedRoute проверяет авторизацию через AppContext и редиректит на /login при отсутствии сессии.
- Layout применяется к приватной части, чтобы все внутренние страницы имели единый каркас.

6) Каркас приложения
Файл src/app/components/Layout.tsx:
- Ant Layout: Sidebar (Sider), Header, Content
- Меню строится динамически по роли пользователя:
  user: главная, избранное, мои заказы
  master: заявки, статистика
  admin: управление услугами, пользователи, статистика
- Адаптивность:
  ширина < 768px -> меню уходит в Drawer
- В шапке — аватар и dropdown с "Профиль" / "Выйти"

7) Глобальное состояние и бизнес‑логика
Файл src/app/context/AppContext.tsx:
Хранит:
- currentUser (текущий пользователь), users, services, reviews, orders, favorites
Инициализация:
- Данные берутся из mockData.ts
- currentUser восстанавливается из localStorage по ключу currentUserId

Основные операции:
- Аутентификация: login, logout, register
- Пользователи: addUser, updateUser, deleteUser
- Услуги: addService, updateService, deleteService
- Отзывы: addReview, deleteReview, likeReview, dislikeReview
  При addReview пересчитывается рейтинг и количество отзывов услуги
- Заказы: addOrder, updateOrder, updateOrderStatus
- Избранное: toggleFavorite
- Расписание:
  getAvailableTimeSlots(date, serviceDuration) — формирует слоты по 1 часу и проверяет
  доступность мастеров с учетом ограничения 10 часов в день

Почему контекст:
- Все страницы и компоненты имеют доступ к одному источнику правды без передачи пропсов по цепочке
- Логика централизована и легко отслеживается

8) Модель данных
Файл src/app/types.ts:
- User (роль: user/master/admin, аватар, дата регистрации, доп. поля рейтинга для мастера)
- Service (цена, длительность, рейтинг, количество отзывов)
- Review (лайки/дизлайки, опциональный deleted)
- Order (статусы, дата, время, связь с мастером)
- Доп. типы для расписаний (TimeSlot, DaySchedule)

9) Мок‑данные
Файл src/app/data/mockData.ts:
- Наборы пользователей, услуг, отзывов и заказов для демонстрации
- Используются при старте как "база данных" в памяти

10) Страницы и их роли
Все страницы лежат в src/app/pages и имеют локальный CSS файл.

Home:
- Список услуг
- Клик по карточке -> /service/:id

Favorites:
- Список избранных услуг
- Пустое состояние при отсутствии избранного

ServiceDetail:
- Детальная страница услуги
- Добавление в избранное
- Список отзывов, лайки/дизлайки, удаление админом
- Форма добавления отзыва (модал)
- Запись на услугу: выбор даты/времени, проверка доступности

Profile:
- Данные пользователя, роль
- Для мастера показываются рейтинг и количество услуг
- Модал смены аватара (демо: генерация случайного изображения)

ManageServices (admin):
- Таблица услуг, добавление/редактирование/удаление
- Форма в модале

UserManagement (admin):
- Таблица пользователей, создание/редактирование/удаление
- Защита от удаления самого себя

Orders (master):
- Список заявок, смена статусов (approved/completed/rejected)

MyOrders (user):
- Список собственных заказов
- Отмена pending заказа

MasterStats / AdminStats:
- Сводные метрики и таблицы по услугам

NotFound:
- 404 страница

11) Связи между страницами и данными
- Все страницы получают данные через useAppContext()
- Навигация управляется router + useNavigate
- Действия (лайк, отмена, одобрение) напрямую вызывают методы контекста

12) Стилизация
- Глобальные стили и переменные: src/styles
- Каждая страница имеет свой CSS файл, подключаемый прямо в TSX
- Tailwind полностью удален
- Ant Design продолжает отвечать за базовые стили UI компонентов

13) Сборка и запуск
Скрипты:
- npm run dev — запуск dev сервера
- npm run build — production сборка

14) Важные ограничения и допущения
- Данные хранятся только в памяти (no backend)
- localStorage используется только для сохранения текущего пользователя
- При перезагрузке страницы данные возвращаются к мокам

15) Как расширять
- Добавить API: заменить mockData + useState на загрузку/мутации через fetch/axios
- Вынести бизнес‑логику в отдельный слой (services) при росте
- Добавить типизацию для API и валидацию
